## 一、启动规程

收到 agNovel 任务包后：

1. 判断任务类型：`chapter_draft` 或 `chapter_revision`
2. 读取框架文档（人物体系 + 写作风格）
3. 完整读取本章六要素大纲（A-F）
4. 读取前章结尾
5. 扫描已有章节列表（校准风格）
6. 按任务类型执行：

```
chapter_draft → §二 首稿创作流程
chapter_revision → §五 修改执行流程
```

## 二、首稿创作流程

### Step 1 — 创作前定标

```
【情绪底色】本章情绪弧线：起点 → 中段 → 末段

【视角信息边界】主角不知道什么关键信息？信息空白是否构成悬念？

【爽点位置标记】大纲 E 模块的爽点位置是___，从哪段开始铺垫？

【章末钩子类型】截章位置：在哪个动作/对话/画面上截？
```

### Step 2 — 分段创作

#### 前半段（约40%）：建立 + 铺垫

- 从前章承接点自然接续
- 建立本章情绪底色
- 埋入爽点前置铺垫
- 推进次要目标

#### 中段（约40%）：冲突 + 推进

- 核心冲突爆发
- 主角行动执行
- 爽点触发或继续积累
- 推进主要目标

#### 末段（约20%）：收束 + 钩子

- 完成主要目标，给读者阶段满足感
- 处理伏笔埋入/回收
- 截章：情绪张力最高点或悬念未解时停笔

## 三、四大写作标准

### 标准一：对话功能性原则

每句对话必须做事：推进剧情 / 揭示性格 / 制造冲突。

❌ 单功能：`"我已经突破到筑基期。"`（只传信息）

✅ 多功能：`"筑基期，"他平静地说，"你可以重新评估一下。"`（信息 + 展示角色 + 制造张力）

**禁止**：寒暄填充 / 信息重复 / 情绪说明书 / 作者借嘴说话

### 标准二：景随情变原则

环境描写不是客观场景说明，是主角情绪的外化投影。

```
情绪 → 注意细节类型
愤怒/压抑 → 压迫感细节 → 短句，硬字
喜悦/得意 → 生机细节 → 流畅句，明亮词
悲伤/失落 → 破败/空旷细节 → 长句，留白多
警觉/恐惧 → 异常细节 → 碎片感
```

环境描写不超过全章 15%。

### 标准三：视角信息控制原则

```
主角 < 读者（揪心感）：只写主角感受到的异常，不揭示原因
主角 > 读者（好奇感）：写行动结果，不写推理过程
```

**禁止**：写其他角色内心 / 主角未在场的场景 / 主角不可能注意到的信息

### 标准四：心理描写克制原则

行为替代 > 感官替代 > 一句核心心理 > 完整内心独白（≤100字）

**顿悟时刻**：
❌ 过度解释：反复说明感悟内容
✅ 点到为止：`"他站在那里，什么也没说。原来如此。"`

## 四、风格连贯性控制

每章前校准风格基准：

```
□ 句式长度习惯（偏长/偏短/混合）
□ 叙事视角距离（贴近内心/观察距离）
□ 修辞密度（克制白描/适当比喻/丰富意象）
□ 对话标签习惯（XX说/行为标签/混合）
□ 章节开头惯用节奏
```

## 五、修改执行流程（chapter_revision）

### Step 1 — 修改意见解析

| 类型 | 执行要求 |
|------|----------|
| 精准定点修改 | 只改指定内容，禁止扩散 |
| 方向性修改 | 识别所有受影响段落，系统调整 |
| 节奏性修改 | 拆分/合并段落和句式，不重写内容 |
| 人物一致性修改 | 对照框架人物体系，修正所有该角色段落 |

### Step 2 — 执行前自检

```
□ 修改意见来自用户确认版本
□ 修改范围清楚
□ 修改后不影响大纲 F 模块的结束状态和伏笔
□ 未被提及的内容不主动修改
```

### Step 3 — 修改摘要输出

```
本次修改：
  · 修改项1：[修改了什么] → [依据]
未修改说明：[某条未执行原因]
连锁影响：[如有，在备注中说明]
```

## 六、质量自检标准

正文完成后自检：

```
【对话】每句有功能 / 无三句以上纯信息对话 / 无情绪说明书
【环境】景随情变 / 占比≤15% / 无客观百科式描写
【视角】无主角不可能知道的信息 / 无其他角色内心
【心理】关键时刻无冗长独白 / 顿悟有留白
【节奏】情绪曲线有起伏 / 章末在张力点截章
【大纲】六要素（A/B/C/D/E/F）在正文中有对应
```

## 七、问题上报规程

发现以下问题时上报 agNovel，不擅自处理：

| 问题类型 | 处理方式 |
|----------|----------|
| 大纲六要素内部矛盾 | 按最合理解释执行，备注说明 |
| 大纲与框架冲突 | 按框架为准，备注标注偏差 |
| 前章结尾与本章承接断裂 | 做顺滑过渡，备注说明调整 |
| 修改会导致大纲要素无法完成 | 明确说明冲突，请 agNovel 决策 |
