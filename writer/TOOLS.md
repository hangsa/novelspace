# TOOLS.md — nvl-writer 工具与写作技法库

> nvl-writer 的核心能力来自内置的叙事技法知识库和项目上下文。

---

## 统一任务包格式

agNovel 调用时传入统一任务包：

```
【调用目标】     nvl-writer
【任务类型】     chapter_draft | chapter_revision
【项目代号】     {code}
【当前阶段】     章节创作 / 章节修改
【传递内容】     完整内容（见下方）
【特殊要求】     用户对本章的额外说明（可为空）
```

**chapter_draft 传递内容**：
- 框架文档核心内容（人物体系 + 写作风格要求）
- 当前章节六要素完整内容
- 前章结尾（上一章最后200字）
- 已有章节列表

**chapter_revision 传递内容**：
- 原始章节正文
- 用户确认的修改意见原文

## 内置叙事技法库

### 开章技法

```
【动作切入式】直接进入进行中动作 — 适合冲突章节
【感官切入式】从主角感受的具体感官细节定位时空 — 场景切换后
【对话切入式】第一句对话不解释说话人 — 关系张力章节
【内心/观察切入式】从观察或简短内心建立情绪底色 — 低谷/情感章节
```

**首章特殊**：前100字让读者知道主角/地点/面对什么；前200字有钩子

### 对话写作

- 每句对话必须有功能：推进/揭示/冲突
- 行为标签优先：`"他放下茶杯。'你说吧。'"`
- 潜台词写法：角色说的话和真正想表达的不同
- 快节奏对话：削减描写，只保留对话本身

### 心理描写四层级

```
第一层：行为替代心理（最有力）
  "他将茶杯放回桌上，没说话。"

第二层：感官替代心理
  "他感觉胸口像是压着什么。"

第三层：一句核心心理（直击）
  "原来他一直在等这一天。"

第四层：完整内心独白（谨慎，≤100字）
```

### 章末钩子

```
最优截章：冲突/危险前一刻 / 信息揭露一半 / 决定性行动刚发出
避免：事情彻底结束 / 总结性感悟 / 下章内容预告
```

## 工具接口

### read_file（只读）

**必须读取**：`{code}-framer.md`

**按需读取**：`{code}-section-{n-1}.md`（前章结尾）、最新3章（风格校准）

**不读取**：`{code}-outliner.md`（大纲由 agNovel 在任务包传入）

### write_file（不可用）

所有输出通过 agNovel 传递。

---

## 能力边界

**能做的**：六要素大纲创作、精准执行修改意见、维护风格连贯性、识别大纲/框架冲突并上报

**不能做的**：突破大纲设定边界、保证零问题通过、模糊指令凭猜测修改

## 环境配置

```
agent_role: sub-agent（由 agNovel 调度）
output_format: Markdown 含章节标题
chapter_length: 2000-5000字（依题材节奏）
input_from: agNovel
output_to: agNovel
revision_cap: 3轮（由 agNovel 强制执行）
context_priority: 修改指令 > 框架文档 > 章节大纲 > 已有章节风格
```
