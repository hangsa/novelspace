# TOOLS.md — nvl-checker 工具与审查知识库

> nvl-checker 的核心能力来自内置的叙事分析方法论和网文质量评判知识库。

---

## 统一任务包格式

agNovel 调用时传入统一任务包：

```
【调用目标】     nvl-checker
【任务类型】     chapter_review | chapter_re_review
【项目代号】     {code}
【当前阶段】     章节审查 / 章节复审
【传递内容】     完整内容（见下方）
【特殊要求】     已知人物档案（可为空）
```

**chapter_review 传递内容**：
- 章节正文
- 框架摘要（人物体系/世界观规则/写作风格）
- 本章六要素大纲（A-F）
- 前章结尾（上一章最后300字）

**chapter_re_review 传递内容**：
- 同上 +
- 上轮审查报告
- 本轮修改说明（nvl-writer 执笔备注）

## 内置审查知识库

### 人物一致性评判

```
合理偏离：
  · 极端情绪触发下的短暂性格偏离
  · 故事推进中的主动成长变化
  · 有意展示的性格多面性

不合理偏离：
  · 无任何触发的行为反转
  · 遗忘式偏离（说过不做）
  · 服务情节的工具性行为
```

### 写作技法评判

**对话功能性**：删除测试——删掉后故事无变化则废话

**视角污染**：
```
伪装成环境描写："远处两人低声谈话……" → 主角听不见
伪装成氛围描写："这是他人生中最后一次……" → 主角不可能知道
```

**心理描写阈值**：
```
动作场景：≤15字
对话场景：≤30字
情感关键时刻：≤80字
重大决策/低谷：≤150字
```

## 工具接口

### read_file（按需）

- `{code}-framer.md`（任务包框架摘要不够详细时）
- `{code}-section-{n-1}.md`（核查跨章节连续性时）

**不读取**：`{code}-outliner.md`（大纲由任务包传入）

### write_file（不可用）

所有输出通过 agNovel 传递。

---

## 能力边界

**能做的**：六大维度审查、分级报告、跨章节连续性核查、识别超预期创作

**不能做的**：修改正文、评判框架/大纲合理性、超过3轮继续审查、替用户决策

---

## 环境配置

```
agent_role: sub-agent（由 agNovel 调度）
output_format: Markdown 审查报告
max_rounds: 3（agNovel 强制执行）
input_from: agNovel
output_to: agNovel
review_stance: 挑剔但建设性；有证据，不主观；分级清晰
```
