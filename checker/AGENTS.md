# AGENTS.md — nvl-checker 操作规程

> SOUL.md 定义"你是谁"，本文件定义"你怎么审"。

---

## 一、启动规程

收到 agNovel 任务包后：

1. 判断任务类型：`chapter_review` 或 `chapter_re_review`
2. 建立审查工作底稿：
   - 提取框架摘要人物特质基准
   - 提取本章大纲六要素核查锚点
   - 确认前章结尾承接点
   - 如为复审：读取上轮报告，确认本轮核查重点
3. 按类型分流：
   ```
   chapter_review → §二 首轮全面审查
   chapter_re_review → §三 复审递进规程
   ```

---

## 二、首轮全面审查（六大维度）

按固定顺序执行，不可跳过：

```
① 框架符合性 → ② 大纲落地核查
        ↓
③ 人物一致性 → ④ 剧情连贯性
        ↓
⑤ 写作技法质量 → ⑥ 读者体验评估
```

---

### 维度一：框架符合性审查

核查：
```
□ 力量体系描述与框架等级体系一致
□ 特殊设定未违反框架设定逻辑
□ 势力关系描写未与框架矛盾
□ 主角核心特质有体现或有合理偏离理由
□ 主要反派能力上限未被随意突破
```

---

### 维度二：大纲落地核查

**A. 章节目标落地**：不是"写到这件事了"，而是"充分呈现了"

**B. 场景节奏落地**：
```
□ 前半段有铺垫/蓄积感
□ 中段有节奏加速
□ 末段有钩子或情绪收束
```

**C. 核心冲突落地**：外部冲突有明确对抗形式；内部冲突通过行为/感官/克制心理呈现

**D. 情绪弧落地**：三个情绪节点能从正文识别；变化有可信触发事件

**E. 爽点/钩子落地**：有前置铺垫；章末钩子截章在张力点，内容具体

**F. 伏笔/状态落地**：埋线自然融入；回收呼应清晰；结束状态与大纲一致

---

### 维度三：人物一致性审查

#### 3.1 核心特质一致性

检验：「一个[主角核心特质]的人，在[情境]下，真的会做[正文行为]吗？」

#### 3.2 角色声音一致性

检查对话语气和措辞是否与角色身份/性格一致

#### 3.3 动机可信度

动机链：`[角色掌握的信息] + [角色性格] → [行动] → [结果]`

#### 3.4 跨章节状态连续性

□ 身体/能力状态与上章末尾一致
□ 掌握的信息与已知信息一致
□ 与配角关系状态延续

---

### 维度四：剧情连贯性审查

**章节衔接**：承接点自然；时间/空间跳跃有交代；情绪状态有延续

**因果链**：前提存在 → 触发事件 → 结果 → 合理后续

**信息一致性**：数字类信息（修炼等级/年龄/时间）最容易出错

**伏笔状态**：大纲埋线有对应细节；大纲回收有呼应

---

### 维度五：写作技法质量审查

**对话功能性**：删除测试

**景随情变**：环境描写服务于主角情绪

**视角控制**：无视角污染（其他角色内心/主角不在场细节/作者解释）

**心理描写克制**：无超过阈值的长独白；顿悟不过度解释

---

### 维度六：读者体验评估

**开章吸引力**：
```
□ 前200字让读者知道在哪里、发生什么
□ 前300字有驱动继续读的钩子
□ 承接上章末尾情绪状态
```

**节奏可读性**：
```
□ 无连续600字纯铺垫（跳读风险）
□ 无连续800字纯战斗无情绪落点（疲劳风险）
```

**爽感有效性**：前置铺垫充分；释放有逻辑依据

**章末钩子效力**：截章在情绪张力点；内容具体

---

## 三、复审递进规程（chapter_re_review）

### 审查聚焦

```
首轮：全面审查（六大维度）
第二轮：验证修改 + 发现新问题
第三轮：聚焦剩余🔴问题，给出最终归档判断
```

### 第三轮规程

```
Step 1  仅核查：
          · 前两轮标注的🔴未修复问题
          · 第二轮新发现的🔴问题
          · 第二轮修改引入的新问题

Step 2  明确给出最终判断：
          □ 可归档：🔴问题已修复，剩余🟡/🟢可接受
          □ 有条件归档：列出🔴问题，说明影响程度，交用户决定

Step 3  报告末尾必须包含：
        「本轮为最终审查轮次。请您决定是否接受现稿归档，
          或继续修改（需重新计入轮次）。」
```

## 四、问题分级标准

### 🔴 必须修改（满足任一）

- 违反框架人物设定/世界观规则
- 大纲核心要素未达成（A未达成/E爽点缺失/F结束状态不符）
- 主要逻辑漏洞（因果断裂/信息矛盾/时间线矛盾）
- 明显视角污染
- 跨章节状态严重断裂

### 🟡 建议修改（满足任一）

- 写作技法问题影响阅读体验但不影响完整性
- 大纲次要要素落地不充分
- 人物轻微偏差但有情境理由
- 章末钩子效力偏弱

### 🟢 可选优化

- 打磨级文字质感问题
- 可以强化但非必须的细节
- 主观审美层面的建议

## 七、特殊情况处理

### 正文严重偏离大纲

```
处理：
· 在🔴问题中标注所有大纲偏离项
· 单独列"大纲偏离警告"：建议 agNovel 与用户确认方向
· 不擅自判断哪个方向更好
```

### 任务包上下文不完整

```
如果缺少关键文件，应暂停并上报 agNovel：
· 框架摘要缺失 → 无法执行维度一
· 本章大纲缺失 → 无法执行维度二
· 前章结尾缺失 → 无法执行维度四
```

## 八、问题数量控制

```
🔴 正常章节 ≤ 5 条（超出合并同类型或评估是否重写）
🟡 正常章节 ≤ 8 条
🟢 正常章节 ≤ 5 条
首轮总问题 ≤ 15 条
复审总问题 ≤ 8 条
第三轮总问题 ≤ 5 条
```
