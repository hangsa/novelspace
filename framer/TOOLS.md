# TOOLS.md — nvl-framer 工具与知识库

> nvl-framer 的核心能力来自内置的网文知识库和结构化设计方法论。
> 本文件说明能力范围和使用规则。

---

## 统一任务包格式

agNovel 调用时传入统一任务包：

```
【调用目标】     nvl-framer
【任务类型】     framework_init | framework_revision
【项目代号】     {code}
【当前阶段】     框架设计 Phase 1 / 框架修改
【传递内容】     完整内容（见下方）
【特殊要求】     用户原话（可为空）
```

**framework_init 传递内容**：
- 用户需求原文
- agNovel 整理的需求摘要

**framework_revision 传递内容**：
- 当前框架版本全文
- 用户修改意见原文

---

## 内置知识库

### 网文类型知识

| 类型 | 关键成功因素 |
|------|------------|
| 修仙/玄幻 | 力量体系自洽 + 晋级节奏 + 矛盾对手梯度 |
| 都市 | 现实感锚点 + 爽感即时性 + 打脸合理性 |
| 无限流 | 规则设计 + 资源稀缺感 + 队伍动态 |
| 历史/架空 | 历史质感 + 谋略逻辑链 + 情感张力 |
| 女频 | 情感颗粒度 + 男女主张力 + 圈层认同 |

### 结构方法论

- 三幕结构（网文化改造版）
- 英雄旅程（用于成长线设计）
- 章末钩子理论（悬念/信息/情感/行动四类）
- 爽点密度控制

## 工具接口

### read_file（只读）

- `{code}-framer.md`（修改迭代时）
- `nvl-research-{date}-{num}.md`（如传入）

**不读取**：正文章节文件、nvl-checker 报告

### write_file（不可用）

不直接写入文件，所有输出通过 agNovel 传递。

### 网络访问（不可用）

不需要实时搜索，框架设计基于内置知识。

---

## 能力边界

**能做的**：设计六模块框架、提出修改建议、识别结构性风险、基于已有框架迭代

**不能做的**：预测爆款、替代大纲/正文/研究、替用户做商业判断

---

## 环境配置

```
agent_role: sub-agent（由 agNovel 调度）
output_format: Markdown
input_from: agNovel
output_to: agNovel
context_priority: 用户原文 > agNovel摘要 > 内置知识库
```
