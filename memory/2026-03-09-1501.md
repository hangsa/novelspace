assistant: 好问题！这是两个不同的概念：

## 多智能体 (Multi-Agent)

指 OpenClaw Gateway 中运行的**多个独立隔离的 agent**。每个 agent：
- 有独立的工作区（`workspace`）
- 独立的认证配置（`agentDir`）
- 独立的人格设定（各自的 `SOUL.md`、`AGENTS.md`）
- 通过 `bindings` 绑定不同的渠道/账户

配置在 `openclaw.json` 中，例如：
```json
{
  agents: { list: [{ id: "home" }, { id: "work" }] },
  bindings: [
    { agentId: "home", match: { channel: "whatsapp", accountId: "personal" } },
    { agentId: "work", match: { channel: "whatsapp", accountId: "biz" } }
  ]
}
```

**本质：多个人共享一个 Gateway 服务器，每个人有独立的"大脑"。**

---

## 子智能体 (Sub-Agent)

指在**一个会话中**动态派生的子任务执行者，用 `sessions_spawn` 调用：
- 临时创建，执行完任务后汇报结果
- 可以指定不同模型、超时时间
- 默认拥有完整工具集（但不能调用会话工具）
- 不能嵌套生成子子智能体

**本质：一个 agent 派"分身"去干活，干完回来汇报。**

---

简单说：
- **多智能体** = 服务器层面，多个独立人格
- **子智能体** = 对话层面，一个 agent 派任务给另一个自己
